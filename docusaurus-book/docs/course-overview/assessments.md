# Assessments

:::tip Project-Based Learning
This course emphasizes hands-on learning through project-based assessments that build practical skills in Physical AI and Humanoid Robotics. Each project is designed to reinforce theoretical concepts with real-world implementation.
:::

## Core Projects

| Project | Description | Technologies Used | Timeline |
|---------|-------------|-------------------|----------|
| ROS 2 Package Development | Design and implement a complete ROS 2 package for robot control | ROS 2, Python/C++, Robot Middleware | Weeks 4-5 |
| Gazebo Simulation Implementation | Create realistic simulation environments for robot testing | Gazebo, Physics Engines, Simulation Frameworks | Weeks 6-7 |
| Isaac-based Perception Pipeline | Build computer vision pipeline for robot perception | Isaac Sim, Computer Vision, Deep Learning | Weeks 8-10 |
| Capstone: Simulated Humanoid Robot | Complete humanoid robot with conversational AI | Full Stack: AI, Robotics, Control, NLP | Weeks 11-13 |

## Assessment Rubric

| Component | Weight | Criteria |
|-----------|--------|----------|
| Technical Implementation | 40% | Code quality, system architecture, proper use of frameworks, performance optimization |
| Problem-Solving | 30% | Challenge identification, creative solutions, iterative improvement, validation |
| Documentation & Presentation | 20% | Technical documentation, project reports, presentation quality, collaboration |
| Innovation & Creativity | 10% | Novel approaches, creative extensions, technology integration, design thinking |

### Detailed Breakdown

#### Technical Implementation (40%)
- Code quality and documentation
- System architecture and design decisions
- Proper use of frameworks and libraries
- Performance optimization

#### Problem-Solving (30%)
- Identification of challenges and constraints
- Creative solutions to technical problems
- Iterative improvement and debugging
- Validation of results

#### Documentation & Presentation (20%)
- Clear technical documentation
- Comprehensive project reports
- Effective presentation of findings
- Peer collaboration and feedback

#### Innovation & Creativity (10%)
- Novel approaches to problem-solving
- Creative extensions beyond basic requirements
- Integration of multiple technologies
- Forward-thinking design choices

## Grading Scale

| Grade | Range | Description |
|-------|-------|-------------|
| A | 90-100% | Exceptional work demonstrating mastery of concepts and innovative thinking |
| B | 80-89% | Good work showing solid understanding and competent implementation |
| C | 70-79% | Adequate work meeting basic requirements with some limitations |
| D | 60-69% | Below expectations with significant gaps in understanding |
| F | Below 60% | Inadequate work failing to meet minimum requirements |

:::note Submission Guidelines
All projects must include source code with proper documentation, technical report explaining design decisions, video demonstration of working system, and performance evaluation and analysis.
:::
